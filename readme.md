## 一些笔记

+ 注意，这个版本（此commit）以及之前的版本都有个巨大的bug，由于由于屏幕的左下角是原点所在位置，所以默认像素的x,y坐标值都是大于0的。**但是，** 当导入模型有部分不在渲染视野中时，会让它们的坐标值为负数，而此时再调用zbuffer来判断深度值时可能会导致越界！也就是 `zbuffer[P.x + P.y * image.get_width()]` 失败。
  > 解决办法：
  >
  >  1. 舍去这些点
  >  2. 确保模型整体都在视野内

+ 这个版本还有一个bug应该是关于透视不对的，以后再改正 :D

+ 这个版本修复了透视不对的问题
